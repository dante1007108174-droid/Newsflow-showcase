## 3. 信源依赖与风险


| 依赖                               | 涉及信源                                 | 风险               | 缓解措施                           |
| -------------------------------- | ------------------------------------ | ---------------- | ------------------------------ |
| RSSHub 镜像（rsshub.rssforever.com） | 量子位、aibase、机器之心、财联社、财新、格隆汇、第驱动之家、爱范儿 | 镜像不可用时 9 个信源同时失效 | 可配置备用 RSSHub 实例                |
| 第三方代理（rss.aishort.top）           | 36氪（日报）                              | 代理不可用时 36氪日报源失效  | 即时查询用的是 36kr.com/feed 直连，可作为备用 |
| HTTP 协议                          | 新华社、中新网                              | 安全风险（非加密传输）      | 目前可用，后续关注是否升级 HTTPS            |

## 4. 已知问题与优化方向


| 问题                   | 影响                                         | 建议                          |
| -------------------- | ------------------------------------------ | --------------------------- |
| 即时查询无专业财经源           | 用户查"财经新闻"时信源单一                             | 从日报的财经源中挑选响应快的补充（如财联社、第一财经） |
| 即时查询无 AI 专业源         | 用户查"AI 新闻"时缺少 aibase、量子位等专业源               | 测试这些源的响应速度，合格则加入即时查询        |
| 36氪即时查询 vs 日报用不同 URL | 即时查询用 36kr.com/feed，日报用 rss.aishort.top 代理 | 统一或确认两个源内容是否一致              |
| 豆包 1.6 即将下架          | 即时查询的关键词扩写模型需替换                            | 提前测试替代轻量模型                  |
