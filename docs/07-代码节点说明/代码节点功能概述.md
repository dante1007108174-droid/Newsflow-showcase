# 代码节点功能概述

> 最后更新：2026-02-28  
> 详细版：`.sisyphus/drafts/代码节点说明手册.md`

---

## 一句话总结

代码节点 = **新闻搬运工**。从各大媒体的 RSS 订阅源抓新闻，清洗、去重、排好序，喂给 AI 做编辑和排版。

---

## 4 个代码节点，各干什么

### 节点① AI和科技主题RSS解析

从 12 个媒体源（IT之家、36氪、量子位、机器之心等）抓取 RSS 新闻，只保留 24 小时内的中文内容，用关键词匹配筛选相关新闻，把不同媒体报道同一件事的重复新闻合并（语义去重），最后控制每个媒体的占比不要太高，输出一份干净的新闻列表给下游 AI 编辑。

AI 主题和科技主题共用这一个节点，靠传入不同的搜索关键词来区分。

### 节点② 财经主题RSS解析

和节点①做的事一样，但换成了 7 个财经媒体源（财联社、财新网、华尔街见闻、第一财经等）。技术上比节点①更成熟：用了多线程并发抓取（更快），网络出错会自动重试。

### 节点③ HTML渲染

接收 AI 编辑好的新闻（分为"核心热点"和"新闻速览"两个板块），生成一封漂亮的 HTML 邮件。包含蓝色渐变头部、根据时间自动切换的问候语（早上好/下午好/晚上好）、卡片式的新闻排版、底部 footer。摘要太长时会智能找句号截断，不会把话截在半句。

### 节点④ 即时查询RSS解析

用户在聊天窗口问"最近有什么AI新闻"时触发。从 8 个综合媒体源并发抓取，不做去重（交给下游 AI 处理），但会用热点词给新闻加权——当前被多家媒体同时报道的话题排名更靠前。比推送版更轻量，追求速度。

---

## 它们之间的关系

```
用户订阅的每日邮件：
  关键词扩写 → 节点①或② → AI编辑新闻 → 节点③渲染HTML → 发邮件

用户实时问新闻：
  关键词扩写 → 节点④ → AI生成摘要 → 直接回复用户
```

## 核心能力一览

| 能力 | 说明 |
|------|------|
| RSS抓取 | 从十几个中文媒体源实时拉取新闻 |
| 中文过滤 | 自动过滤掉英文内容，只保留中文 |
| 时效控制 | 只要最近24小时的新闻 |
| 关键词匹配 | 标题命中权重高，描述命中权重低 |
| 语义去重 | 不同媒体报道同一件事→合并，记录热度 |
| 信源均衡 | 防止某一家媒体霸屏，控制占比上限 |
| 热点词提取 | 统计当前高频词，辅助排序 |
| HTML邮件生成 | 生成适配各邮件客户端的响应式邮件 |
