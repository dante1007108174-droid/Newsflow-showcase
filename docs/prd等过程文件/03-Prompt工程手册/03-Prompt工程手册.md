# Prompt 工程手册

本手册记录了 NewsFlow.ai 智能助手的 Prompt（系统提示词）设计原则、演进历程及维护规范。

## 1. 设计原则

智能助手的 Prompt 设计遵循以下核心原则：

- **身份一致性**：明确助手作为 "AI每日简报" 专属助手的身份，专注于新闻获取与订阅管理。
- **上下文驱动**：利用前端注入的 `[System Context: user_id="..."]` 实现无感知的用户识别，避免主动向用户索取私密 ID。
- **意图强约束**：通过详尽的“意图识别表”和“技能模块”说明，确保 LLM 能够准确映射用户需求到对应的工作流。
- **参数槽位化**：将所有技能调用参数（如 `action`, `keyword`, `new_email`）明确定义，并规定其获取来源和必填性。
- **安全与隐私**：内建防注入逻辑，严禁泄露 `user_id` 或跨账户操作。

## 2. 提示词结构分析

当前系统提示词（位于 `coze_workflow/agent系统提示词.txt`）采用模块化结构，共 146 行。其核心架构如下：

### 2.1 核心规则（身份与上下文）
规定了助手必须从消息末尾自动提取 `user_id` 并严禁主动提问。这是实现多端数据同步的关键。

### 2.2 意图识别表 (Intent Table)
通过 Markdown 表格显式定义了关键词与技能的映射关系：
- **查看新闻** → `get_instant_news_using`
- **查询订阅** → `subscribe_using (action=查询订阅)`
- **新增/修改订阅** → `subscribe_using (action=修改订阅)`
- **取消订阅** → `subscribe_using (action=取消订阅)`
- **立即推送** → `send_mail_using`

### 2.3 技能模块说明
针对每个工作流提供了详细的参数说明和交互流程示例。例如，在“订阅管理”模块中，区分了“新用户订阅”、“查询”、“修改”、“取消”四种场景。

### 2.4 对话规范与模板
定义了简洁友好的语言风格，并提供了欢迎语、主题询问、成功提示等快捷回复模板。

## 3. 演进历程

### 早期阶段：Bugfix 驱动
在项目早期，曾维护过 Prompt 迭代记录表；但随着个人同时承担产品、评测、前后端与工作流落地，记录逐渐减少。该阶段 Prompt 更新以 Bugfix 为主，主要目标是把核心链路跑通(意图识别、槽位补齐、工具调用)。

### 后期阶段：两个重大版本演进
在前后端与工作流逐步稳定后，系统提示词做了两次“结构级”调整：

- **v1 版本**：实现了基本的意图识别与技能路由，但**缺失 `user_id` 注入/强制提取规则**。在订阅/推送等需要绑定用户身份的场景里，容易出现 `input_user_id` 传递不稳定或需要额外追问的信息缺口(同时也带来隐私风险)。
- **v2 版本（当前版本）**：
    - **新增 `user_id` 提取规则**：配合前端在消息末尾注入 `[System Context]` 的逻辑，强制助手进行提取并静默处理。
    - **引入意图识别表与示例**：为了增强稳定性，增加了显式的意图映射表和多场景交互示例（One-shot/Few-shot），显著提升了槽位填充（Slot-filling）的准确率。

## 4. 关键技术细节

### 参数填充可靠性 (Slot-filling)
为了确保 `keyword`（主题）的准确性，Prompt 规定：
- `keyword` 无默认值，必须让用户主动选择。
- 提供了模糊表述映射（如“大模型”映射为“AI”），增强了容错性。

### 温度设置 (Temperature Rationale)
在路由评估和意图识别中，建议将 Temperature 设置为 **0.5**。
- **原因**：0.5 在保持回复多样性的同时，能有效抑制模型在技能选择时的随机性，确保逻辑判断的严谨度优于纯创意写作。

### 提示词注入防御
- 助手被赋予明确的“隐私保护”规则，禁止向用户展示其内部 `user_id`。
- 在逻辑上限制了助手只能通过 `input_user_id` 操作当前上下文对应的数据。

## 5. 维护清单 (Checklist)

未来修改 Prompt 时，请务必检查以下各项：

- [ ] **变量一致性**：新增技能参数是否与 `backend/routes/workflow.js` 中的调用逻辑一致？
- [ ] **上下文提取**：是否依然保留了对 `[System Context]` 的强制提取指令？
- [ ] **映射表更新**：新增的功能是否已同步更新至“意图识别表”？
- [ ] **交互示例**：是否为新功能提供了至少一个正向和反向的交互示例？
- [ ] **隐私自检**：新指令是否可能导致 `user_id` 或 `email` 在回复中泄露？
