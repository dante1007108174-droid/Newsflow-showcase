# Daily AI News 埋点说明

> 本文档描述项目当前的埋点系统：已埋哪些点、触发时机、数据结构和上报方式。
> 最后更新：2026-03-10

---

## 1. 埋点系统概述

本系统采用 **「前后端分离」** 的设计：
- **前端事件**：由前端页面通过 `/api/track` 接口主动上报
- **后端事件**：由后端服务在处理业务逻辑时自动写入 Supabase
- **存储**：所有事件统一写入 `public.events` 表

### 1.1 身份标识

| 字段 | 生成方式 | 存储位置 | 用途 |
|------|----------|----------|------|
| `user_id` | `crypto.randomUUID()` 或回退时间戳随机数 | `localStorage['daily_ai_news_user_id']` | 长期用户标识 |
| `session_id` | 同上 | `sessionStorage['daily_ai_news_session_id']` | 单次会话标识 |

**注意**：埋点身份（`daily_ai_news_*`）与业务身份（`cozeUserId`）目前是两套独立体系。

## 2. 前端埋点事件

前端通过 `frontend/js/app.js` 中的 `trackEvent()` 函数上报，接口为 `POST /api/track`。

### 2.1 已启用事件

#### `page_view` - 页面访问

- **触发时机**：页面 `DOMContentLoaded` 事件
- **代码位置**：`frontend/js/app.js:102`
- **上报数据**：

```json
{
  "event_name": "page_view",
  "event_data": {
    "url": "/",
    "referrer": "",
    "ua": "Mozilla/5.0 ..."
  }
}
```

#### `client_error` - 客户端错误

- **触发时机**：
  - JavaScript 运行时错误（`window.onerror`）
  - 未处理的 Promise 拒绝（`unhandledrejection`）
- **代码位置**：`frontend/js/app.js:70-93`
- **上报数据**：

```json
{
  "event_name": "client_error",
  "event_data": {
    "error_type": "window_error",
    "message": "错误信息",
    "source": "文件名.js",
    "line": 123,
    "stack": "堆栈信息..."
  }
}
```

#### `chat_trigger_click` - 浮窗 Bot 点击

- **触发时机**：用户点击主页右下角浮窗 Bot 按钮，并从关闭状态打开聊天窗时
- **代码位置**：`frontend/index.html:1128`（`toggleChat()` 的打开分支）
- **用途**：
  - 统计用户主动点击浮窗入口的次数
  - 衡量聊天入口的吸引力和点击率
  - 分析用户访问浮窗 Chat 的频次，评估 Chat 作为信息入口的价值
- **上报数据**：

```json
{
  "event_name": "chat_trigger_click",
  "event_data": {
    "entry": "floating_bot",
    "surface": "home",
    "device_type": "mobile | desktop",
    "open_state": "closed_to_open"
  }
}
```

#### `chat_guide_click` - 聊天引导语点击

- **触发时机**：用户点击浮窗聊天欢迎区中的 3 条引导语之一时
- **代码位置**：`frontend/index.html:2409`（`handleGuideClick()`）
- **用途**：
  - 统计用户更偏好哪类引导语
  - 判断浮窗入口的实际需求方向
  - 了解用户主要使用 Chat 的场景（看新闻、查订阅、测试邮件）
- **当前引导语**：
  - `news_24h`：`看看24小时内的AI新闻`
  - `subscription_info`：`查看我的订阅信息`
  - `email_test`：`邮件效果测试`
- **上报数据**：

```json
{
  "event_name": "chat_guide_click",
  "event_data": {
    "guide_key": "news_24h",
    "guide_text": "看看24小时内的AI新闻",
    "surface": "chat_welcome",
    "device_type": "mobile | desktop"
  }
}
```

### 2.2 白名单限制

前端只能上报以下白名单事件（定义在 `backend/lib/tracking.js:14-18`）：

```javascript
const FRONTEND_TRACK_EVENTS = new Set([
  'page_view',
  'chat_trigger_click',
  'chat_guide_click',
  'message_sent',
  'client_error',
]);
```

**注意**：`message_sent` 目前在白名单中，但主前端页面 `index.html` 并未实际调用上报。

## 3. 后端埋点事件

后端通过 `backend/lib/tracking.js` 中的 `trackServerEvent()` 函数写入，所有事件自动携带 `user_id` 和 `session_id`（如果请求中提供了）。

### 3.1 已启用事件

#### `subscribe` - 订阅事件

- **触发接口**：`POST /api/subscription/upsert`
- **代码位置**：`backend/routes/subscription.js:203-217`
- **触发时机**：订阅创建、更新、或未变更时
- **说明**：当前记录的是“订阅处理结果”，不是“点击立即订阅按钮”本身；只有后端成功处理后才会写入
- **上报数据**：

```json
{
  "event_name": "subscribe",
  "event_data": {
    "action": "create | update | unchanged",
    "keyword": "ai | 财经 | 科技",
    "has_email": true,
    "replaced": false,
    "unchanged": false
  }
}
```

#### `feedback` - 反馈事件

- **触发接口**：`POST /api/feedback`
- **代码位置**：`backend/routes/feedback.js:163-174`
- **触发时机**：用户提交点赞或点踩后
- **上报数据**：

```json
{
  "event_name": "feedback",
  "event_data": {
    "feedback_type": "like | dislike",
    "reasons": ["irrelevant", "outdated", "too_long"],
    "has_comment": true
  }
}
```

#### `workflow_call` - Coze 工作流调用

- **触发接口**：`POST /api/workflow/send-test`
- **代码位置**：`backend/routes/workflow.js:92-104`（成功）、`136-150`（失败）
- **触发时机**：发送测试邮件触发 Coze 工作流后
- **上报数据**：

```json
{
  "event_name": "workflow_call",
  "event_data": {
    "channel": "send_test",
    "workflow_id": "xxx",
    "status": "success | failed",
    "duration_ms": 1234,
    "http_status": 200,
    "error_code": null
  }
}
```

#### `email_sent` - 邮件发送

- **触发接口**：`POST /api/workflow/send-test`
- **代码位置**：`backend/routes/workflow.js:106-117`（成功）、`152-164`（失败）
- **触发时机**：工作流调用成功后记录邮件发送状态
- **上报数据**：

```json
{
  "event_name": "email_sent",
  "event_data": {
    "keyword": "AI",
    "success": true,
    "email_domain": "qq.com",
    "http_status": 200,
    "error_code": null
  }
}
```

### 3.2 允许但未启用事件

以下事件在后端允许列表中，但目前没有任何代码实际触发：

| 事件名 | 定义位置 | 说明 |
|--------|----------|------|
| `message_sent` | `backend/lib/tracking.js:5` | 用户发送聊天消息 |
| `bot_reply` | `backend/lib/tracking.js:6` | 机器人回复消息 |

## 4. 事件数据结构

所有事件统一写入 Supabase `public.events` 表，结构如下：

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | UUID | 主键，默认 `gen_random_uuid()` |
| `user_id` | TEXT | 用户标识（来自前端 localStorage） |
| `session_id` | TEXT | 会话标识（来自前端 sessionStorage） |
| `event_name` | TEXT | 事件名称 |
| `event_data` | JSONB | 事件元数据 |
| `created_at` | TIMESTAMPTZ | 服务端写入时间 |

### 4.1 安全约束

- **敏感字段拦截**：`event_data` 中禁止包含以下 key（大小写不敏感）：
  - `email`、`message_content`、`raw_message`
  - `token`、`authorization`、`password`、`secret`、`api_key`
- **大小限制**：`event_data` JSON 序列化后不超过 4096 bytes
- **降级处理**：埋点写入失败不会影响主业务流程，返回 202 状态码

## 5. API 接口

### 5.1 前端上报接口

```
POST /api/track
```

请求头：
```
Content-Type: application/json
X-User-Id: <user_id>
X-Session-Id: <session_id>
```

请求体：
```json
{
  "event_name": "page_view",
  "event_data": {
    "url": "/"
  }
}
```

限流：60 次/分钟/IP

### 5.2 业务接口（自动埋点）

以下接口在处理业务时会自动触发埋点，无需前端额外调用：

| 接口 | 方法 | 触发的事件 |
|------|------|-----------|
| `/api/subscription/upsert` | POST | `subscribe` |
| `/api/feedback` | POST | `feedback` |
| `/api/workflow/send-test` | POST | `workflow_call`、`email_sent` |

## 6. 主页用户行为埋点清单

在主页上，以下用户行为会被埋点记录：

| 用户行为 | 事件名 | 触发时机 | 记录数据 |
|----------|--------|----------|----------|
| **打开主页** | `page_view` | 页面加载完成 | URL、来源、设备信息 |
| **点击浮窗 Bot** | `chat_trigger_click` | 点击右下角浮窗按钮打开聊天 | 入口来源、设备类型、打开状态 |
| **点击引导语** | `chat_guide_click` | 点击聊天欢迎区的 3 条引导语之一 | 引导语 key、引导语文案、设备类型 |
| **提交订阅** | `subscribe` | 订阅成功处理后 | 操作类型(create/update/unchanged)、主题、邮箱状态 |
| **反馈评价** | `feedback` | 点赞/点踩成功后 | 反馈类型、原因、是否带评论 |
| **前端错误** | `client_error` | JS 报错或 Promise 未处理 | 错误类型、信息、堆栈 |

**暂未记录的行为**（如需可后续补充）：
- 主题选择切换
- 邮箱输入框聚焦
- 聊天消息发送/回复
- 聊天窗口关闭
- 页面滚动深度

## 7. 快速测试

### 7.1 测试浮窗 Bot 点击埋点

打开主页 → 点击右下角浮窗 Bot 按钮 → 查看 `events` 表：

```sql
SELECT * FROM events 
WHERE event_name = 'chat_trigger_click' 
ORDER BY created_at DESC 
LIMIT 5;
```

### 7.2 测试引导语点击埋点

打开浮窗聊天 → 点击任意引导语 → 查看 `events` 表：

```sql
SELECT * FROM events 
WHERE event_name = 'chat_guide_click' 
ORDER BY created_at DESC 
LIMIT 5;
```

### 7.3 测试页面访问埋点

```bash
curl -X POST "http://localhost:3000/api/track" \
  -H "Content-Type: application/json" \
  -H "X-User-Id: test_user" \
  -H "X-Session-Id: test_session" \
  -d '{"event_name":"page_view","event_data":{"url":"/","referrer":""}}'
```

### 6.2 测试错误埋点（带敏感字段会被拦截）

```bash
curl -X POST "http://localhost:3000/api/track" \
  -H "Content-Type: application/json" \
  -d '{"event_name":"page_view","event_data":{"email":"test@test.com"}}'

# 预期返回：400 "event_data 包含敏感字段"
```
