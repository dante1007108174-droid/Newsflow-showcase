# LLM Agent Badcase 合集管理（通用）

当你在做 router/dispatcher/agent/workflow 质量评测时，用这个 skill 维护“失败用例合集”（badcase collection）：
- 失败用例可追溯（证据路径 + Run ID）
- 根因可聚合（便于批量修复、避免重复踩坑）
- 状态可追踪（待修复/待观察/已修复/不修复）

## 触发语句
- “把最新 badcase 同步到合集”
- “整理失败用例并归因”
- “更新 badcase 合集表头/框架”
- “从 report.json 批量导入失败用例”

## 必要输入
- badcase 合集路径（Excel），例如 `docs/badcase合集（更新中）.xlsx`
- 失败来源（至少一个）：
  - 评测报告 JSON 路径（如 `tests/reports/**/agent_e2e_report_*.json`）
  - 或人工指定的失败用例 ID + 失败原因 + 证据
- 本轮运行信息：Run ID、Bot/Agent ID、时间戳（用于备注与可追溯）

## Badcase 合集：表头与框架（强约束）

推荐的通用表头（单 Sheet，第一行为表头）：

`ID | 问题名称 | 问题描述 | 根因 | 影响 | 优化 | 问题状态 | 备注`

字段约定：
- `ID`：唯一标识。推荐：`E041` / `D016` / `NEWS-01` 这种可读、稳定的 ID。
- `问题名称`：一句话概括问题（便于目录检索）。
- `问题描述`：包含“触发输入/场景 + 失败表现 + 失败频次（可选）”。
- `根因`：一句话技术归因（优先可行动）。
- `影响`：对用户/指标/验收的影响。
- `优化`：可执行的修复建议（prompt/代码/数据/平台侧）。
- `问题状态`：推荐枚举：`待修复` / `待观察` / `已修复` / `不修复`。
- `备注`：证据链接（报告路径/截图/Run ID/时间），以及回归结果摘要。

兼容映射（历史文件可能出现的列名差异）：
- `名称` -> `问题名称`
- `优化方式` / `建议` -> `优化`

## 更新策略（避免把合集搞乱）

### 去重与更新规则
- 以 `ID` 作为主键。
- 如果 `ID` 已存在：
  - 仅更新 `问题状态` 与 `备注`（追加本轮证据），除非你确认根因/描述需要更正。
- 如果 `ID` 不存在：
  - 追加新行，必须填写 8 列，不允许留空核心字段（至少：ID/问题名称/问题描述/根因/问题状态/备注）。

### 备注追加规则（保留历史）
- `备注` 采用追加而不是覆盖：`<旧备注>; <新备注>`。
- 新备注至少包含：`YYYY-MM-DD` + `Run ID` + `报告路径或证据`。

### 合并/压缩（可选）
当大量用例共享同一根因时，可以在合集里增加一个“Group 行”（例如 `GROUP-xx`）：
- `问题描述` 写“共性总结 + 代表性输入列表（2-5 条）”。
- 原子用例仍可保留或仅保留一部分代表性 ID。

## 根因分类（通用 taxonomy）

先分层：
- `infra`：超时/网络/平台瞬态错误/限流/发布延迟
- `format`：输出契约不一致、解析器提取错误、字段缺失
- `logic`：意图路由/决策错误、槽位优先级错误、错误调用工具
- `quality`：内容质量不达标（幻觉/不完整/不符合要求）

再细分（可选）：
- `memory`：历史记忆干扰导致追问/补槽逻辑偏离（典型：缺参追问没问）
- `tool`：工具/工作流参数字段变更、别名映射错误

## 建议工作流（从报告到合集）

1. 读取报告 JSON，抽取失败用例（`是否通过 == false`）。
2. 对每个失败用例生成 badcase 条目：
   - `ID`：用例 ID
   - `问题名称`：按失败类型生成（例如“D3_工作流执行异常”）
   - `问题描述`：拼接“测试场景 + 用户输入 + 失败原因（截断）”
   - `根因`：先按 `infra/format/logic/quality` 分类，再写一句话
   - `优化`：给出最小修复建议（不要泛泛而谈）
   - `问题状态`：新发现默认 `待修复` 或 `待观察`
   - `备注`：写 Run ID + 报告路径
3. 写入 Excel：按主键更新或追加。
4. 回归通过后：将 `问题状态` 更新为 `已修复`，备注追加回归证据。

## 反模式
- 没有证据路径（报告/截图/日志）就写 badcase
- 一个 badcase 条目塞多个互不相关的根因
- 只写“模型不稳定/平台问题”而不给可验证的下一步
- 覆盖备注导致历史证据丢失
